# AI Frontend Architecture Template (Angular / React / Next.js)

This repository is a **starter template powered by AI agents** designed to accelerate the creation of scalable frontend applications using:

- Angular
- React
- Next.js

It provides a structured **agent-based development system** that generates, enforces, and maintains clean architecture automatically.

---

# Purpose

This project is not a finished application.

It is a **development framework + AI agent system** that helps you:

- Scaffold new projects faster
- Enforce consistent architecture
- Maintain high code quality
- Automate feature generation
- Ensure performance and test coverage

---

# Supported Frameworks

## Angular
- Standalone components
- Signals-first state management
- Facade pattern architecture
- Strict DI boundaries
- Feature-based modular structure

## React
- Hook-based architecture
- Feature isolation
- Service + hook separation
- Client-first design (optional Next.js hybrid support)

## Next.js
- Server Components by default
- Server Actions for mutations
- Hybrid server/client architecture
- Performance-first rendering strategy

---

# AI Agent System

This repository includes a full **AI development pipeline**:

## 1. Planner Agents
- Break down product requirements into features
- Define domain boundaries
- Generate task structures

## 2. Init Agents
- Bootstrap project architecture
- Create folder structure
- Configure base tooling
- Enforce architectural rules

## 3. Feature Agents
- Generate full features (services, state, UI, tests)
- Maintain clean architecture per framework rules

## 4. Quality Agents
- Refactor code
- Enforce architecture rules
- Optimize performance
- Validate dependency structure

## 5. Test Agents
- Generate unit, integration, and E2E tests
- Enforce coverage requirements

# How to Use

## 1. Choose a stack
Select one:
- Angular
- React
- Next.js

---

## 2. Run the Planner Agent
Provide a product description:

> Example: "Championship management system for teams and athletes"

The planner will:
- define features
- generate domain model
- create task breakdown

---

## 3. Run Init Agent
Initializes project structure:

- folders
- base architecture
- rules enforcement
- tooling setup

---

## 4. Run Feature Agents
Generate features such as:
- authentication
- teams
- athletes
- dashboards

Each feature includes:
- UI
- state
- services
- tests
- validation

---

## 5. Maintain with Quality Agents
Use:
- refactor agent → improve structure
- test agent → enforce coverage
- performance agent → optimize runtime
- QA agent → validate architecture
